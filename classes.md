# 3. 클래스별 상세 (업데이트판)

> 기준 파일: `data.js` · `state.js` · `player.js` · `cigarette.js` (최신 버전)
> 이전 문서(`game_doc.md`)에서 **바뀐 점**은 각 표 아래 ⚠ 메모로 표시.

**읽는 법**

- **DATA** = 정적(플레이 중 안 변함). **STATE** = 런타임(변함).
- DATA 표 컬럼: `속성 | 값 | 도메인 | 설명`
- STATE 표 컬럼: `필드 | 타입 | 정적/런타임 | 초기값 | 설명`
- 🔒 = 여러 상태를 한 번에 바꾸는 트랜잭션 함수.

## Player (player)

`player.js` 의 `Player` 클래스 인스턴스. `STATE.player` 에 한 개만 들어간다(`resetGameState` 에서 `new Player()`).

**DATA.CONFIG.PLAYER** — 정적 설정

| 속성   | 값                                     | 도메인 | 설명                          |
| ------ | -------------------------------------- | ------ | ----------------------------- |
| BOX_W  | 360                                    | >0     | 충돌 박스 너비(px)            |
| BOX_H  | 360                                    | >0     | 충돌 박스 높이(px)            |
| SPEED  | 4                                      | >0     | 프레임당 이동 픽셀            |
| SPRITE | `img_assets/characters/99_default.png` | 경로   | 기본 이미지 (index.html 기준) |

**STATE.player** — 런타임 (Player 인스턴스, 단일)

| 필드         | 타입                               | 정적/런타임  | 초기값                          | 설명                                                            |
| ------------ | ---------------------------------- | ------------ | ------------------------------- | --------------------------------------------------------------- |
| x            | number                             | 런타임       | `MAP.WIDTH/2 − BOX_W/2` (=2020) | 맵(월드) 좌우 위치. `walk()` 가 변경                            |
| y            | number                             | 런타임(고정) | `VIEWPORT.GROUND_Y` (=40)       | 바닥 y. 이번 스코프 고정(세로 안 움직임)                        |
| looking      | `'left' \| 'right'`                | 런타임       | `'right'`                       | 바라보는 방향. render 가 좌우 반전에 사용                       |
| playerStatus | `'idle' \| 'picking' \| 'smoking'` | 런타임       | `'idle'`                        | 행동 상태(플레이어 FSM). 현재는 항상 `'idle'`(애니 다음 스코프) |

**Player 클래스 메서드**

| 함수        | 읽기(reads)                            | 쓰기(writes)   | 🔒  | 설명                                                               |
| ----------- | -------------------------------------- | -------------- | --- | ------------------------------------------------------------------ |
| `walk(dir)` | `MAP.WIDTH`, `PLAYER.BOX_W·SPEED`, `x` | `x`, `looking` |     | `dir`(-1/0/+1)만큼 이동 후 `[0, MAP.WIDTH−BOX_W]` clamp; 방향 갱신 |
| `getBox()`  | `x`, `y`, `PLAYER.BOX_W·BOX_H`         | (없음)         |     | 충돌 판정용 `{x, y, w, h}` 반환. **순수 조회**                     |

| ------- | ------ | ---------- | ---- | ---- | ------------------------------ |

| cigar_m | 10     | 45%        | 70   | 30   | `img_assets/items/cigar_m.png` |
| cigar_l | 20     | 10%        | 90   | 30   | `img_assets/items/cigar_l.png` |

**STATE.cigarettesArray[]** — 런타임 (인스턴스 배열)

| 필드       | 타입                                  | 정적/런타임          | 초기값·생성처                    | 설명                            |
| ---------- | ------------------------------------- | -------------------- | -------------------------------- | ------------------------------- |
| id         | string                                | 런타임(생성 후 고정) | spawn 발급 (`"cig_0"`…)          | DOM 매칭용 고유 id              |
| type       | `'cigar_s' \| 'cigar_m' \| 'cigar_l'` | 런타임(생성 후 고정) | spawn 시 결정                    | 종류                            |
| points     | number                                | 런타임(고정)         | `= CIGARETTE_TYPES[type].points` | 점수(종류에서 복사). smoke 사용 |
| boxW, boxH | number                                | 런타임(고정)         | `= CIGARETTE_TYPES[type]`        | 히트박스(종류에서 복사)         |
| sprite     | string                                | 런타임(고정)         | `= CIGARETTE_TYPES[type].sprite` | 이미지 경로. render 사용        |
| x          | number                                | 런타임               | spawn 시 배치(미지정 시 맵 랜덤) | 위치                            |
| y          | number                                | 런타임(고정)         | `GROUND_Y + BOX_H − boxH`        | 위치(밑면이 바닥선에 맞음)      |
| collected  | bool                                  | 런타임               | `false`                          | 주웠으면 true → 렌더 제외       |

**Cigarette 클래스 메서드**

| 함수        | 읽기(reads)              | 쓰기(writes) | 🔒  | 설명                                           |
| ----------- | ------------------------ | ------------ | --- | ---------------------------------------------- |
| `collect()` | (없음)                   | `collected`  |     | 주운 것으로 표시(true)                         |
| `getBox()`  | `x`, `y`, `boxW`, `boxH` | (없음)       |     | 충돌 판정용 `{x, y, w, h}` 반환. **순수 조회** |

## LungGauge (lungGauge)

**DATA.CONFIG** — 게이지 정적 값

| 속성                  | 값                        | 도메인 | 설명                                                                |
| --------------------- | ------------------------- | ------ | ------------------------------------------------------------------- |
| LUNG_GAUGE_MAX        | 100                       | >0     | 최대치 / 엔딩 임계값                                                |
| LUNG.SPRITE           | `/img_assets/ui/lung.png` | 경로   | 폐 이미지                                                           |
| LUNG.WIDTH            | 120                       | >0     | 표시 너비(px)                                                       |
| LUNG.HEIGHT           | `(379·120)/420` ≈108.3    | >0     | 폐 이미지 비율(420×379) 유지한 높이                                 |
| LUNG.FILL_TOP_PERCENT | 22                        | 0~100  | 채움이 멈추는 위쪽 천장(%). 기관지는 안 채움(시각용)                |
| LUNG.SEGMENTS         | 20                        | >0     | 게이지를 몇 조각(계단)으로 나눠 채울지. **순수 시각용**(MAX와 독립) |

**STATE.lungGauge** (실제 변수명 `STATE.playerLungGauge`)

| 필드            | 타입   | 정적/런타임 | 초기값 | 설명                                                   |
| --------------- | ------ | ----------- | ------ | ------------------------------------------------------ |
| playerLungGauge | number | 런타임      | 0      | 도메인 `[0, LUNG_GAUGE_MAX]`. `addGauge()` 에서만 변경 |

> ⚠ 변경점: 변수명이 `lungGauge` → **`playerLungGauge`**. 시각용 `LUNG` 설정(SEGMENTS·FILL_TOP_PERCENT 등)이 새로 추가됨 — 게이지 **값**(100)과는 독립이니 헷갈리지 말 것.

## Scene (scene)

**DATA.CONFIG.SCENE** — 영상(이미지) 씬 지속 시간

| 속성          | 값  | 도메인 | 설명                         |

**STATE.scene** (실제 변수명 `STATE.currentScene`)

## Input (input)

**DATA.CONFIG.KEYS** — 키 바인딩

| 속성     | 값                    | 도메인 | 설명              |
| -------- | --------------------- | ------ | ----------------- |
| left     | `['arrowleft', 'a']`  | 키목록 | 왼쪽 이동(홀드)   |
| right    | `['arrowright', 'd']` | 키목록 | 오른쪽 이동(홀드) |
| interact | `['e']`               | 키목록 | 줍기(탭)          |

**STATE.input**

| 필드            | 타입 | 정적/런타임 | 초기값 | 설명                                                          |
| --------------- | ---- | ----------- | ------ | ------------------------------------------------------------- |
| left            | bool | 런타임      | false  | ← 눌려 있나(홀드)                                             |
| right           | bool | 런타임      | false  | → 눌려 있나(홀드)                                             |
| interactDown    | bool | 런타임      | false  | E 가 지금 눌려 있나(홀드). 엣지 감지의 "직전 상태" 역할       |

## 4. 다음 단계: 수도코드(pseudo code) 작성표 — 템플릿

아래 함수들은 **로직이 아직 비었거나(TODO) 새로 들어올 자리**다. 각 함수마다 같은 형식의 표를 하나씩 두고, 주니어가 의사코드를 채우게 한다.

### 작성표 컬럼(공통 형식)

| 단계 | 입력(reads) | 출력(writes/return) | 수도코드(한 줄씩) | 비고          |
| ---- | ----------- | ------------------- | ----------------- | ------------- |
| 1    | …           | …                   | 만약 ~라면 …      | 경계조건 주의 |
| 2    | …           | …                   | …                 | …             |

> 규칙: **읽기/쓰기를 먼저 적고**, 그다음 의사코드를 단계별로. 트랜잭션이면 변경 순서 근거를 비고에 남길 것.

### 어디에 어떤 표를 둘지 (배치 제안)

| 함수(자리)                  | 파일/위치            | 무엇을 채우나                                                                  | 현재 상태                          |
| --------------------------- | -------------------- | ------------------------------------------------------------------------------ | ---------------------------------- |
| `pickRandomCigaretteType()` | `spawn.js`           | `percentage`(45/45/10) **가중치**를 반영한 랜덤 종류 뽑기                      | ⚠ 지금은 균등(1/3) — 확률 TODO     |
| `pickRandomDirection()`     | `spawn.js`(신규)     | 담배 `direction` `'left'\|'right'` 50:50 랜덤 뽑기(개발문서의 방향 필드)       | ❌ 아직 없음 — 인스턴스에 필드부터 |
| 랜덤 `SPAWN.INTERVAL`       | `spawn.js`           | 고정 90 대신 범위(예: 60~120) 안에서 매번 랜덤 간격 뽑기                       | ⚠ data.js 에 TODO 명시             |
| `spawnOneCigarette(x)`      | `spawn.js`           | 위치/종류/방향 결정 → 인스턴스 push 까지의 전체 절차                           | 일부 구현(방향·확률 빠짐)          |
| `updateCamera()`            | `camera.js`(신규)    | `EDGE_MARGIN` 데드존 기반 스크롤: 플레이어가 끝 근처면 `camera.x` 갱신 + clamp | ❓ loop.js 가 호출하나 본체 미첨부 |
| `smoke()` (재진입 금지)     | `interaction.js`     | picking/smoking 상태가 부활하면 `playerStatus !== 'idle'` 조건 추가            | 🔒 트랜잭션, 주석 처리된 자리 있음 |
| 줍힌 담배 재생성            | `spawn.js`/`smoke()` | 주운 담배를 3~10초 뒤 랜덤 종류로 되살릴지(개발문서)                           | ⚠ 미확정(BLOCKING)                 |

### 예시: `pickRandomDirection()` 작성표 (빈칸 채우기용 샘플)

| 단계 | 입력(reads) | 출력(return)        | 의사코드(한 줄씩)                  | 비고                       |
| ---- | ----------- | ------------------- | ---------------------------------- | -------------------------- |
| 1    | (난수)      | —                   | `r ← 0~1 난수`                     |                            |
| 2    | `r`         | `'left' \| 'right'` | `r < 0.5 면 'left' 아니면 'right'` | 50:50, 추후 비율 조정 가능 |

> 같은 형식으로 위 표의 각 함수마다 한 표씩 만들어 두면, 주니어는 "읽기/쓰기 → 단계" 순서로만 채우면 됨.
